# 文件处理全能助手 - 用户操作手册

## 1. 系统简介

文件处理全能助手是一个分布式文档处理系统，支持以下核心功能：

- **格式转换**: PDF/Word/Excel/PPTX/MD/SVG/PNG 之间的相互转换
- **PDF处理**: 合并、分割、提取文本/图片、转图片
- **批量处理**: 支持大规模文件批量转换
- **高并发**: 支持100+并发请求

## 4. 使用指南

### 4.1 打开应用

在浏览器中访问 http://localhost 即可打开文件处理全能助手。

### 4.2 上传文件

1. 点击页面中央的拖放区域，或点击"选择文件"
2. 选择需要处理的文件（支持PDF、Word、Excel、PPTX、PNG、Markdown等格式）
3. 文件上传后会自动显示在文件列表中

### 4.3 选择转换格式

文件上传成功后，在下拉菜单中选择目标格式：
- PDF → Word、PDF → 图片
- Word → PDF
- Excel → PDF
- PPTX → PDF
- PNG → PDF
- Markdown → PDF
- SVG → PNG、SVG → PDF

### 4.4 开始转换

1. 点击"开始转换"按钮
2. 转换进度会实时显示
3. 转换完成后，状态会变为"转换完成"

### 4.5 下载结果

转换完成后，点击"下载"按钮即可获取转换后的文件。

## 5. 批量处理

### 5.1 批量上传

可以一次上传多个文件，系统会自动为每个文件创建独立的转换任务。

### 5.2 批量转换

点击"开始全部转换"按钮，系统会依次处理队列中的所有文件。

## 6. API接口使用

### 6.1 健康检查

```bash
curl http://localhost:8000/health
```

### 6.2 获取支持的格式

```bash
curl http://localhost:8000/api/v1/formats
```

### 6.3 单文件转换

```bash
curl -X POST "http://localhost:8000/api/v1/convert?conversion_type=word_to_pdf" \
  -F "file=@your_file.docx"
```

### 6.4 批量转换

```bash
curl -X POST "http://localhost:8000/api/v1/convert/batch?conversion_type=word_to_pdf" \
  -F "files=@file1.docx" \
  -F "files=@file2.docx"
```

### 6.5 查看任务状态

```bash
curl http://localhost:8000/api/v1/tasks/<task_id>
```

### 6.6 下载转换结果

```bash
curl -o output.pdf http://localhost:8000/api/v1/tasks/<task_id>/download
```

### 6.7 PDF合并

```bash
curl -X POST http://localhost:8000/api/v1/pdf/merge \
  -H "Content-Type: application/json" \
  -d '{"file_ids": ["file_id_1", "file_id_2"], "output_filename": "merged.pdf"}'
```

### 6.8 PDF分割

```bash
curl -X POST http://localhost:8000/api/v1/pdf/split \
  -H "Content-Type: application/json" \
  -d '{"file_id": "file_id", "page_ranges": ["1-2", "5-5"]}'
```

### 6.9 PDF文本提取

```bash
curl -X POST http://localhost:8000/api/v1/pdf/extract \
  -H "Content-Type: application/json" \
  -d '{"file_id": "file_id", "extract_text": true}'
```

## 8. 故障排查

### 8.1 服务无法启动

1. 检查Docker是否运行：
   ```bash
   docker ps
   ```

2. 检查端口是否被占用：
   ```bash
   lsof -i :8000
   ```

3. 查看错误日志：
   ```bash
   docker-compose -p file-processor logs
   ```

### 8.2 转换失败

1. 检查Celery Worker是否运行：
   ```bash
   docker ps | grep celery
   ```

2. 查看Worker日志：
   ```bash
   docker-compose -p file-processor logs celery_conversion
   ```

3. 检查Redis连接：
   ```bash
   docker exec -it file-processor-redis-1 redis-cli ping
   ```

### 8.3 前端无法访问

1. 检查Nginx是否运行：
   ```bash
   docker ps | grep nginx
   ```

2. 查看Nginx日志：
   ```bash
   docker-compose -p file-processor logs nginx
   ```

### 8.4 任务一直处理中

1. 检查任务队列：
   ```bash
   docker exec -it file-processor-redis-1 redis-cli LLEN celery
   ```

2. 查看Worker状态（Flower）：
   访问 http://localhost:5555

## 9. 性能优化

### 9.1 增加Worker数量

修改 `docker-compose.yml` 中Celery服务的 `replicas` 数量：

```yaml
celery_conversion:
  deploy:
    replicas: 4  # 增加worker数量
```

### 9.2 调整并发数

修改启动命令中的 `--concurrency` 参数：

```yaml
command: celery -A celery_config worker --loglevel=info --concurrency=8 -Q conversions
```

## 11. 常见问题

### Q1: 如何修改上传文件大小限制？
修改 `backend/app/main.py` 中的配置，或在Nginx中设置client_max_body_size。

### Q2: 如何添加新的转换格式？
1. 在 `backend/app/services/converter.py` 中添加转换方法
2. 在 `backend/app/tasks/convert_tasks.py` 中添加任务处理
3. 在前端 `frontend/src/App.tsx` 中添加格式选项

### Q3: 转换超时怎么办？
增加Celery任务超时时间，或优化转换逻辑。
